CreateInclusionOutline(String, Uri)

Creates a new OpmlOutline that represents an inclusion outline using the supplied parameters.

Declaration
public static OpmlOutline CreateInclusionOutline(string text, Uri url)
Parameters
Type Name Description
System.String text

The textual content of the outline.
Uri url

A that represents an http address.
Returns
Type Description
OpmlOutline

A new OpmlOutline object that represents an inclusion outline, initialized using the supplied parameters.
Remarks

When a outline is expanded in an outliner, if the url ends with .opml, the outline expands in place. This is called inclusion.

If the url does not end with .opml, the link is assumed to point to something that can be displayed in a web browser.

The difference between link and include is that link may point to something that is displayed in a web browser, and include always points to an OPML file.

This method will create an OpmlOutline with a ContentType of include if the url ends with .opml, otherwise the ContentType will have a value of link.

Type Description

A new OpmlOutline object that represents a subscription list outline, initialized using the supplied parameters.
Remarks

A subscription list is a possibly multiple-level list of subscriptions to feeds. Each sub-element of the body of the OPML document is a node of type rss or an outline element that contains nodes of type rss.

Today, most subscription lists are a flat sequence of rss nodes, but some aggregators allow categorized subscription lists that are arbitrarily structured. A validator may flag these files, warning that some processors may not understand and preserve the structure.
